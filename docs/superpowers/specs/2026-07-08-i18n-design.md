# i18n 国际化支持

**日期**：2026-07-08
**范围**：为插件增加多语言消息系统，覆盖所有玩家可见消息（约 400 处 `MessageUtil.sendMessage/broadcast` 调用）

---

## 背景

当前所有玩家可见消息为 Java 代码内硬编码中文字符串（如 `MessageUtil.sendMessage(player, "&c你还没有岛屿！")`）。服务器无法切换语言，无法服务非中文玩家。

`NameTranslator` 已用 `Component.translatable` 处理原版方块/物品/实体（依赖客户端语言），但插件自有消息不走该路径。

本 spec 引入 `LanguageManager` + `MessageUtil.send(key, args)` API，将字面量迁移到 YAML 语言文件，支持服务器级 locale 切换。

### 不在范围内

- TrMenu 菜单文件（`skyblockmenu/*.yml`）文本 — 维持现状
- 任务内容文件（`tasks/Chapter*/*.yml`）— 维持现状
- `plugin.yml` 中 `description`/`usage` 字段 — Bukkit 元数据，不参与运行时消息
- PAPI 占位符返回值 — 数据型返回（数字、岛屿名等），不含可翻译文本
- 控制台日志（`MessageUtil.consolePrint/consoleWarn/consoleError`）— 保留字面量
- `MessageUtil.sendMessage(sender, String)` 旧方法 — 保留，用于控制台诊断等场景

## 1. 架构概览

### 新增组件

```
team.starm.starmskyblock.message
├── MessageUtil.java        (扩展，新增 key-based API)
└── LanguageManager.java     (新增 — 加载/缓存/查询消息)
```

### 资源文件

```
src/main/resources/
└── messages/
    └── messages_zh_CN.yml  (默认语言文件，由现有 Java 字面量提取而来)
```

服务器运行时位置：`plugins/StarMSkyblock/messages/messages_xx_XX.yml`，用户可在此添加新语言文件。

### 启动序列变更

在 `StarMSkyblock.onEnable()` 第 2 步（6 个 ConfigManager 初始化）之后、第 3 步（提取 schematics）之前插入：

```
2.5. 初始化 LanguageManager
     - 从 config.yml 读取 locale 字段（默认 zh_CN）
     - 加载 messages/messages_<locale>.yml
     - 缺失则回退到内置 zh_CN
     - 再缺失则记录错误并禁用插件
```

### config.yml 新增字段

```yaml
# i18n 设置
locale: 'zh_CN'   # 当前激活的语言代码，需对应 messages/messages_<locale>.yml 文件
```

### 调用方迁移前后对比

```java
// 之前
MessageUtil.sendMessage(player, "&c你还没有岛屿！");
MessageUtil.sendMessage(player, "&c岛屿重命名冷却中，请等待 &e" + remaining + " &c秒后再试！");

// 之后
MessageUtil.send(player, "island.no-island");
MessageUtil.send(player, "island.rename.cooldown", Map.of("remaining", remaining));
```

### 兼容性

- 保留 `MessageUtil.sendMessage(sender, String)` 旧方法 — 作为字面量发送的 API 入口，供未来内部诊断、调试或 PAPI 扩展等场景使用。**当前约 400 处调用全部迁移到 `send(key)` 后，此方法在代码库中暂时无调用方**，但保留 API 表面以便后续需要时直接复用
- 新方法 `MessageUtil.send(sender, key)` / `send(sender, key, Map)` 是 i18n 入口
- `MessageUtil.colorize/stripColor/parse/consolePrint/consoleWarn/consoleError` 保持现状
- `NameTranslator` 不动 — 它依赖客户端语言，与服务器 i18n 体系正交

## 2. 键命名与文件结构

### 键命名约定

层级点分式，按「模块.子模块.语义」组织，最多 4 层：

```yaml
general:
  no-permission: "&c你没有权限执行此操作！"
  player-only: "&c该命令只能由玩家执行。"

island:
  no-island: "&c你还没有岛屿！"
  create:
    success: "&a岛屿创建成功！正在传送..."
    failed: "&c岛屿创建失败：{reason}"
  rename:
    usage: "&c用法: /is rename <新名称>"
    cooldown: "&c岛屿重命名冷却中，请等待 &e{remaining} &c秒后再试！"
    too-long: "&c岛屿名称不能超过 &e{max} &c个字符（颜色代码不计入）！"
    success: "&a岛屿名称已修改为: &e{name}"
    failed: "&c修改失败，请稍后重试。"
    no-permission: "&c你没有权限修改岛屿名称！"
  delete:
    success: "&a岛屿已删除。"
    no-more-deletes: "&c你已经没有删除次数了。剩余次数：{remaining}"

upgrade:
  radius:
    max-reached: "&c岛屿半径已达上限。"
    insufficient-funds: "&c你的余额不足。需要 &e{cost}&c，当前余额 &e{balance}&c。"

task:
  submit:
    no-progress: "&c该任务尚未完成，无法提交。"
    success: "&a任务提交成功！"

command:
  is:
    usage: "&c用法: /is <create|spawn|border|help>"
```

**约定**：

1. 键全小写，单词用连字符 `kebab-case`（`no-permission`、`too-long`、`insufficient-funds`）
2. 层级最多 4 层：`<模块>.<子模块>.<语义>`，超出说明子模块该拆分
3. 占位符统一用 `{name}` 命名形式（不用 `{0}` 位置式）
4. 颜色代码 `&a/&c/&e` 等保留在 YAML 值中，由 `MessageUtil` 现有颜色管线解析

### 文件结构

```
plugins/StarMSkyblock/messages/
├── messages_zh_CN.yml   (内置，随 jar 解压)
└── messages_xx_XX.yml   (用户自行添加，如 en_US)
```

- 一个 locale = 一个文件。一个文件 = 一个完整语言包。
- 不支持跨文件引用。所有键必须出现在当前激活 locale 文件中。
- 启动时把内置 `messages/messages_zh_CN.yml` 解压到插件数据目录（仅在目标文件不存在时覆盖，保留用户修改）。

### 占位符替换

`LanguageManager.format(key, Map<String, ?>)` 实现：

```java
public String format(String key, Map<String, ?> args) {
    String template = resolve(key);  // 返回 key 对应的 YAML 值（含颜色代码）
    if (args == null || args.isEmpty()) return template;
    for (var e : args.entrySet()) {
        template = template.replace("{" + e.getKey() + "}", String.valueOf(e.getValue()));
    }
    return template;
}
```

- 简单字符串替换，不引入 `MessageFormat`（避免 `'` 转义、`{0,date}` 等复杂语法干扰颜色代码）
- 未匹配占位符保留原样（`{unknown}`），便于译者发现拼写错误
- 替换发生在颜色解析之前 — 占位符值不会被当作颜色代码解析

## 3. 加载与运行时数据流

### 启动加载流程

```
StarMSkyblock.onEnable()
  ├─ ... 前置步骤 ...
  ├─ ConfigManager 初始化（含新字段 locale）
  ├─ LanguageManager.initialize(plugin)
  │    ├─ 1. 读取 config.yml 的 locale 字段（默认 zh_CN）
  │    │    - 验证格式：匹配 ^[a-z]{2}_[A-Z]{2}$ (如 zh_CN, en_US)
  │    │    - 不匹配则 consoleWarn 并回退 zh_CN
  │    ├─ 2. 解压内置 messages/messages_zh_CN.yml 到 plugins/StarMSkyblock/messages/
  │    │    - 仅在目标文件不存在时覆盖（保留用户修改）
  │    ├─ 3. 加载 plugins/StarMSkyblock/messages/messages_<locale>.yml
  │    │    - 不存在 → 回退内置 jar 内 messages/messages_<locale>.yml
  │    │    - 都不存在 → 回退 zh_CN，consoleWarn
  │    │    - zh_CN 都不存在 → 禁用插件，consoleError 报告关键资源缺失
  │    ├─ 4. YamlConfiguration.loadConfiguration(...) 解析
  │    │    - YAML 语法错误 → consoleError 报告行号，回退 zh_CN
  │    └─ 5. 扁平化为 Map<String, String>（key 全路径 → 字符串值）
  │         - 用 yaml.getKeys(true) 取所有叶节点
  │         - 跳过 section 节点（仅保留 string 叶）
  │         - 存入 ConcurrentHashMap（不可变快照）
  └─ ... 后续步骤 ...
```

### 扁平化示例

YAML：
```yaml
island:
  rename:
    cooldown: "&c请等待 &e{remaining} &c秒"
```

扁平化后：
```java
Map<String, String> {
    "island.rename.cooldown" -> "&c请等待 &e{remaining} &c秒"
}
```

`LanguageManager.format(key, args)` 用扁平 map O(1) 查询。

### 运行时查询路径

```
调用方:
  MessageUtil.send(player, "island.rename.cooldown", Map.of("remaining", 120))
       │
       ▼
MessageUtil.send(sender, key, args):
  1. silent 检查（player + SILENT_PLAYERS 集合）→ 静默则直接返回
  2. String template = languageManager.format(key, args)
  3. String colored = ColorUtils.toColor(template)  // 解析 & 代码为 §
  4. sender.sendMessage(colored)
       │
       ▼
LanguageManager.format(key, args):
  1. String raw = messages.get(key)
  2. raw == null → 控制台 consoleWarn（每 key 仅一次）+ 返回 key 本身
  3. 替换占位符 {name} → 值
  4. 返回 raw
```

### 重载命令

扩展已有 `/isadmin reload`（`ReloadCommand.java`），新增一行：

```java
plugin.getLanguageManager().reload();
```

重载流程：

1. 重新读取 `config.yml`（locale 字段可能变更）
2. 重新加载 `messages/messages_<locale>.yml`
3. 替换 `volatile` map 引用（旧引用仍可被正在执行的查询使用）

### 静态访问 LanguageManager

`MessageUtil` 是静态工具类，需要在启动时注入 `LanguageManager` 引用：

```java
public class MessageUtil {
    private static volatile LanguageManager languageManager;

    public static void setLanguageManager(LanguageManager lm) {
        languageManager = lm;
    }

    public static void send(CommandSender sender, String key) {
        send(sender, key, null);
    }

    public static void send(CommandSender sender, String key, Map<String, ?> args) {
        if (sender instanceof Player p && SILENT_PLAYERS.contains(p.getUniqueId())) return;
        if (languageManager == null) {
            sender.sendMessage(key);  // 防御：未初始化时返回 key 文本
            return;
        }
        sender.sendMessage(colorize(languageManager.format(key, args)));
    }
}
```

`StarMSkyblock.onEnable()` 启动期调用 `MessageUtil.setLanguageManager(languageManager)` 注入。

### 线程安全

- `messages` Map 用 `ConcurrentHashMap`，初始化后只读
- `reload()` 用新 map 替换旧引用（`volatile` 字段），不修改旧 map
- 已发送的查询不阻塞，重载过程中可能短暂命中旧或新 map — 可接受

## 4. 错误处理与边界情况

### 启动期错误

| 场景 | 处理 |
|---|---|
| `config.yml` 中 `locale` 字段缺失 | 使用默认值 `zh_CN`，`consoleWarn` 提示 |
| `locale` 格式非法（如 `chinese`） | 回退 `zh_CN`，`consoleWarn` 提示期望格式 `xx_XX` |
| 内置 `messages/messages_zh_CN.yml` 在 jar 中缺失 | 启动失败：`consoleError` + `getServer().getPluginManager().disablePlugin(this)` |
| 用户 locale 文件（如 `messages_en_US.yml`）缺失 | 回退内置 `zh_CN`，`consoleWarn` 提示文件路径 |
| YAML 语法错误 | `consoleError` 报告文件 + 行号，回退 `zh_CN`；若 `zh_CN` 也错则禁用插件 |
| 启动期 `LanguageManager.initialize` 抛异常 | 捕获并禁用插件（i18n 是核心基础设施，缺失则后续 `MessageUtil.send` 全部返回 key 文本，用户体验已严重退化） |

### 运行期错误

| 场景 | 处理 |
|---|---|
| `MessageUtil.send(sender, "nonexistent.key")` | `LanguageManager.format` 内部检测到 `messages.get(key) == null` 时，调用 `MessageUtil.consoleWarn("i18n 缺失键: " + key)`，**每 key 每会话仅一次**（用 `LanguageManager` 内部的 `ConcurrentHashMap.newKeySet` 跟踪），并返回字面 key 文本给 `MessageUtil.send` 输出给玩家 |
| `MessageUtil.send(sender, null)` | 静默返回（与现有 `sendMessage(sender, null)` 一致） |
| `sender` 为 `null` | 抛 `IllegalArgumentException`（与现有一致 — 现有 `sendMessage` 用 `@NotNull`） |
| 占位符值 `null` | 替换为字符串 `"null"`（与 `String.valueOf(null)` 一致），不抛异常 |
| 占位符值含 `{` 或 `}` 字符 | 直接替换 — 替换是单遍左到右，不递归（避免注入风险） |
| 模板含 `{` 但无 `}` | 原样输出（不视为占位符） |
| 重载时另一线程正在查询 | 用 `volatile` map 引用 + `ConcurrentHashMap`：旧引用仍可用，新引用原子替换 |

### 占位符替换的边界情况

模板：`"&c坐标: &e{x}&c, &e{y}&c, &e{z}"`
args: `{x=10, y=20, z=30}`

替换顺序无关（按 `Map.entrySet()` 顺序遍历，但每个占位符键唯一），结果：`"&c坐标: &e10&c, &e20&c, &e30"`。

不存在的占位符 `{unknown}` 保留原样 — 译者拼写错误在测试时极易发现。

### 静默模式交互

`MessageUtil.SILENT_PLAYERS` 集合记录带 `-s` 标志的玩家。新 `send(sender, key)` 同样要先查静默集合 — 复用现有逻辑。

### 日志输出

- 启动期问题 → `consoleError`/`consoleWarn`（带 LOG_PREFIX）
- 缺失 key 警告 → `consoleWarn("i18n 缺失键: " + key)`
- 缺失 key 警告**不**走 `consoleError`，因为这是运行期可恢复问题，不应污染错误日志
- 重载成功 → `consolePrint("i18n 已重载：locale=" + locale)`

## 5. 迁移计划

### 模块分组与执行顺序

按现有目录结构分批迁移，每批可独立提交：

| 批次 | 范围 | 实际调用点 |
|---|---|---|
| 1 | `command/subcommand/*` 全部子命令 | 269 |
| 2 | `command/IslandCommand.java` + `AdminCommand.java` + `IslandPermissionCommand.java` (dispatch 层) | 66 |
| 3 | `task/*` (TaskManager + TaskCommand) | 31 |
| 4 | `level/*` (LevelManager 等) | 16 |
| 5 | `island/*` (InvitationManager + IslandCreateTask + IslandDeleteTask + IslandManager + IslandSerializer) | 12 |
| 6 | `permission/BasePermissionManager.java` + `setting/manager/*` + `listener/*` 等散落点 | 6 |
| - | 合计 | 400 |

### 迁移单元操作（每个 `sendMessage` 调用）

**Before**:
```java
MessageUtil.sendMessage(player, "&c岛屿重命名冷却中，请等待 &e" + remaining + " &c秒后再试！");
```

**Step 1**：在 `messages/messages_zh_CN.yml` 对应模块下新增条目
```yaml
island:
  rename:
    cooldown: "&c岛屿重命名冷却中，请等待 &e{remaining} &c秒后再试！"
```

**Step 2**：把字符串拼接改为命名占位符
```java
MessageUtil.send(player, "island.rename.cooldown", Map.of("remaining", remaining));
```

**Step 3**：保留必要的 `import java.util.Map;`（如未引入）

### 迁移约束

1. **不修改业务逻辑**：仅替换消息输出方式，分支、循环、状态变更保持不变
2. **不重命名变量**：占位符名与局部变量名一致（`remaining` → `{remaining}`）
3. **保留颜色代码**：所有 `&c`、`&e`、`<gradient:...>` 等 ColorUtils 标签原封不动移入 YAML
4. **逐批提交**：每批一次提交，提交信息形如 `refactor(i18n): migrate island/* to message keys`
5. **不批量替换**：每个调用点单独审视，确认占位符语义和上下文（避免把 `"&c" + x` 误改为 `Map.of("x", x)`，因为颜色代码不属于占位符值）

### 控制台日志的边界

- `MessageUtil.consolePrint/consoleWarn/consoleError` 调用**不迁移** — 保留字面量
- 例外：`ReloadCommand` 中的成功提示是面向管理员发送者的，应迁移
- 启动期诊断（如「WorldEdit 未安装」）不迁移

## 8. 关键决策记录

| 决策 | 选择 | 理由 |
|---|---|---|
| i18n 范围 | 仅插件消息 | 菜单/任务是数据型内容，多语言维护成本高 |
| Locale 来源 | 全局统一 | 简化实现，符合项目当前需求 |
| 文件格式 | YAML | 与现有 ConfigManager 风格一致 |
| 内置语言 | 仅 zh_CN | 其他语言由用户自行添加 |
| 迁移方式 | 一次性全量 | 接口干净，避免长期两条代码路径 |
| 键命名 | 最多 4 层 kebab-case | 兼顾可读性与扁平化查询效率 |
| 占位符 | 命名式 `{name}` | 自解释，比 `{0}` 友好 |
| 缺失 key 兜底 | 返回字面 key 文本 + 一次性警告 | 玩家能识别问题，调试容易 |
| 控制台日志 | 保留字面量 | 启动期诊断不依赖 i18n |
| LanguageManager 访问 | `MessageUtil` 静态注入 | 调用简短，与现有 SILENT_PLAYERS 设计一致 |
| 重载命令 | 扩展现有 `/isadmin reload` | 不新增命令，符合用户习惯 |
